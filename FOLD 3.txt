# FOLD 3 — MANIFESTO / POSICIONAMENTO | Adega Própria Landing Page

---

## Role & Design Philosophy

You are a senior UI designer and frontend developer building the **manifesto fold** of the Adega Própria landing page — the brand's soul section. This is NOT a feature list. This is NOT a "what we do" section. This is a **declaration** — the kind of editorial composition you'd find in a Tenuta Luce or Kettmeir website, where the brand speaks in its own voice about what it stands for.

This fold is a **pause for soul, not conversion.** There is no CTA here (CTAs live in Folds 1, 4, 6, 8, 9). The job of this fold is to build emotional depth and brand credibility — to make the visitor slow down, feel the weight of the proposition, and believe that Adega Própria is the real thing.

Build a premium, **wine-dark** editorial manifesto that sells the brand's philosophy: origin, exclusivity, label personalization, and winery credibility. Four declarations, each poetic, each visually weighted, arranged in an asymmetric editorial composition that feels like a luxury magazine spread — not a SaaS landing page.

**The darkness is wine-red, not green.** Canvas is `#1A0F10`. Shadow layer is `#2A1416`. Warm depths are `#8C3D20` and `#A62C21`. Gold and beige whisper only as text, italic emphasis, thin decorative lines, and precious metadata. Never green. Never bright. Never loud.

**Reference energy for this specific fold:**
- **Kettmeir** — typographic elegance, the way text and photography coexist with generous whitespace
- **Tenuta Luce** — italic serif restraint, values-driven declarations ("Rispetto per la terra, per la bellezza, per le persone")
- **Pasqua** — manifesto boldness, chapter-based storytelling energy

**Design mandates (non-negotiable):**
- No emoji icons. No generic gradients. No inline styles (except for complex radial gradients Tailwind cannot express).
- No placeholder images except dark, moody Unsplash wine photography with heavy dark overlay treatments.
- Typography must be intentional — continue the Fraunces + DM Sans pairing established in Fold 1.
- Every element must feel like it belongs in an Awwwards-winning editorial.
- Mobile-first: design for 375px, then enhance for desktop.
- The fold must pass the "screenshot test" — if posted on Dribbble, it receives genuine engagement.

## Animation & Motion Specification

This fold uses **scroll-triggered entrance animations** (unlike Fold 1's load-triggered entrance). Elements animate in as they enter the viewport, creating a narrative pace as the visitor reads the manifesto.

### Scroll-Triggered Pattern

Use Framer Motion's `useInView` hook on a ref for each animated element:

```jsx
const ref = useRef(null);
const isInView = useInView(ref, { once: true, amount: 0.3 });
```

`amount: 0.3` means the animation triggers when 30% of the element is in view. `once: true` ensures it only animates once per page visit.

Apply via:
```jsx
<motion.div
  ref={ref}
  initial={{ opacity: 0, y: 40 }}
  animate={isInView ? { opacity: 1, y: 0 } : { opacity: 0, y: 40 }}
  transition={{ duration: 0.9, ease: [0.22, 1, 0.36, 1] }}
>
```

The easing `[0.22, 1, 0.36, 1]` is the project's signature cubic-bezier — consistent with Fold 1.

### Element-by-Element Animation

| Element | Initial | Animate (on inView) | Duration | Delay |
|---|---|---|---|---|
| "— MANIFESTO —" label | `opacity: 0, y: 20` | `opacity: 1, y: 0` | 700ms | 0ms |
| Master headline | `opacity: 0, y: 40` | `opacity: 1, y: 0` | 1000ms | 150ms |
| Intro prose | `opacity: 0, y: 25` | `opacity: 1, y: 0` | 800ms | 350ms |
| Short hairline | `scaleX: 0` | `scaleX: 1` | 900ms | 550ms |
| **Per declaration (scroll-triggered individually):** | | | | |
| Image | `opacity: 0, scale: 1.05` | `opacity: 1, scale: 1` | 1200ms | 0ms |
| Roman numeral | `opacity: 0, x: -20` | `opacity: 1, x: 0` | 700ms | 100ms |
| Declaration title | `opacity: 0, y: 30` | `opacity: 1, y: 0` | 800ms | 200ms |
| Declaration body | `opacity: 0, y: 20` | `opacity: 1, y: 0` | 700ms | 350ms |
| Metadata line | `opacity: 0` | `opacity: 1` | 600ms | 550ms |
| **Closing declaration (IV):** | | | | |
| "IV" numeral | `opacity: 0, y: 20` | `opacity: 1, y: 0` | 700ms | 0ms |
| Closing italic prose | `opacity: 0, y: 40` | `opacity: 1, y: 0` | 1100ms | 200ms |
| Signature "— Adega Própria" | `opacity: 0, y: 15` | `opacity: 1, y: 0` | 700ms | 500ms |

### Image Scale Effect (subtle parallax)

Each editorial image has a subtle scale-in effect. On inView, the image scales from `1.05` to `1.0` over 1.2 seconds — creating the feeling that the images are settling into place rather than popping in. This is more cinematic than a hard fade.

### Subtle Scroll Parallax on Images (desktop only)

On desktop (`lg:` and up), each image receives a very gentle vertical parallax on scroll — not aggressive, just enough to create depth:

```jsx
const imgRef = useRef(null);
const { scrollYProgress } = useScroll({
  target: imgRef,
  offset: ["start end", "end start"]
});
const imgY = useTransform(scrollYProgress, [0, 1], ["0%", "-8%"]);
```

Apply `imgY` to the `<motion.img>` as `style={{ y: imgY }}`. Only apply when `!isMobile && !prefersReducedMotion`. The `-8%` travel is subtle — the image shifts up slightly as the visitor scrolls past, creating a sense of dimensional depth.

### `prefers-reduced-motion` Respect

If `prefersReducedMotion` is true, skip all entrance animations (render at final state), disable parallax, and disable scale effects.

```jsx
const prefersReducedMotion = typeof window !== 'undefined' 
  ? window.matchMedia('(prefers-reduced-motion: reduce)').matches 
  : false;
```

### Image Treatment (CRITICAL)

Each of the three editorial images must receive heavy treatment to remain wine-dark and on-brand:

**Image wrapper structure:**
```jsx
<figure className="relative overflow-hidden aspect-[4/5] lg:aspect-[3/4] bg-[#2A1416]">
  <motion.img
    src={IMAGE_URL}
    alt="..."
    className="absolute inset-0 w-full h-full object-cover brightness-[0.5] lg:brightness-[0.55]"
    loading="lazy"
    style={{ y: imgY }}  // desktop parallax only
  />
  {/* Tint overlay — keeps images firmly in wine territory */}
  <div 
    className="absolute inset-0 pointer-events-none mix-blend-multiply" 
    style={{ background: 'linear-gradient(135deg, rgba(26,15,16,0.4) 0%, rgba(140,61,32,0.25) 100%)' }}
  />
  {/* Bottom gradient — ensures visual separation from ambient background */}
  <div 
    className="absolute inset-x-0 bottom-0 h-1/3 pointer-events-none" 
    style={{ background: 'linear-gradient(to top, rgba(26,15,16,0.5) 0%, transparent 100%)' }}
  />
  {/* Thin border to define the frame */}
  <div className="absolute inset-0 ring-1 ring-inset ring-[#BF8C60]/8 pointer-events-none" />
</figure>
```

**Image brightness:** `brightness-[0.5]` on mobile, `brightness-[0.55]` on desktop. Images should feel integrated into the wine-dark atmosphere, never bright or stock-photo-like.

**Tint overlay:** The diagonal gradient `rgba(26,15,16,0.4) → rgba(140,61,32,0.25)` with `mix-blend-multiply` pushes the images firmly into wine territory, preventing any color drift toward green or generic dark.

**Aspect ratio:** `aspect-[4/5]` on mobile (portrait editorial feel), `aspect-[3/4]` on desktop (slightly wider, fits the editorial grid).

**Thin border:** `ring-1 ring-inset ring-[#BF8C60]/8` — a barely-visible gold hairline around each image frame, adding editorial polish.

## Micro-interactions

This fold is primarily for reading, not interacting — but the few interactive touches should feel precious.

### Image Hover (desktop only)

Very subtle — the image should feel like it's breathing, not animating aggressively:

- `hover:brightness-[0.65]` — the tiniest brightness lift (from 0.55 baseline)
- `transition-all duration-1000 ease-out` — slow, luxurious
- The inner `<motion.img>` can also scale subtly: `group-hover:scale-[1.02]` within a `group` parent figure
- No shadow changes, no sudden color shifts — this is editorial restraint, not product-card interaction

### Text Selection Styling

Add a custom selection style that matches the wine-dark palette (this detail elevates the whole fold):

```css
::selection {
  background-color: rgba(191, 140, 96, 0.25);
  color: #D9BBA9;
}
```

Include this inside the `<style>` block at the top of the component. When a visitor highlights manifesto text to copy or share, the selection color is on-brand gold instead of the default browser blue.

### Numbered Marker Hover (desktop)

The roman numerals (I, II, III, IV) become slightly brighter on hover of their containing article:
- Default: `text-[#BF8C60]/60`
- On `group-hover` (article parent): `group-hover:text-[#BF8C60]/90`
- `transition-colors duration-700`
